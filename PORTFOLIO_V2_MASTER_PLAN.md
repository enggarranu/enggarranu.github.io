# Executive Summary

## Project Name

Executive Technical Portfolio V2

---

## Purpose

This project aims to build a premium personal website that represents Enggar Ranu Hariawan as a senior engineering professional specializing in large-scale database platforms, migration architecture, reliability engineering, and platform engineering.

This website is **NOT** intended to be a traditional CV, online resume, or generic developer portfolio.

Instead, it should communicate engineering maturity, technical leadership, architectural thinking, ownership, and measurable business impact.

The website should immediately demonstrate that the owner is capable of designing, leading, and operating mission-critical database platforms at enterprise scale.

# Chapter 02 — Design Philosophy

---

# Overview

This chapter defines the design philosophy of the Executive Technical Portfolio V2.

The goal is not simply to create a visually attractive website.

The goal is to communicate engineering maturity, technical leadership, reliability, and architectural thinking through design.

Every visual element should reinforce the perception that the owner builds production platforms for enterprise-scale systems.

The design should remain timeless, clean, lightweight, and maintainable.

---

# Design Objectives

The website should achieve the following objectives.

1. Create a strong first impression within the first 10 seconds.

2. Clearly communicate engineering identity before employment history.

3. Guide visitors through an intentional engineering narrative.

4. Demonstrate measurable impact before technologies.

5. Reflect senior engineering maturity rather than visual complexity.

6. Prioritize readability over decoration.

7. Remain relevant for at least the next five years.

---

# Design Principles

The following principles guide every design decision.

---

## Simplicity

Every element must have a purpose.

Avoid unnecessary visual decorations.

Avoid excessive colors.

Avoid visual noise.

Whitespace is considered a design element.

The layout should feel effortless to read.

---

## Engineering First

This website is not a marketing website.

It is an engineering portfolio.

Engineering content should always take priority over visual effects.

Visual design should support the story rather than dominate it.

---

## Impact Before Technology

Visitors should first understand:

What was achieved.

Only afterwards should they discover:

How it was achieved.

Never begin a section by listing technologies.

Always begin with the engineering challenge and measurable outcome.

---

## Platform Thinking

The website should consistently communicate platform engineering rather than operational work.

Replace phrases like:

"I migrated databases."

With:

"I designed migration platforms enabling repeatable production migrations."

Replace:

"I monitored databases."

With:

"I established monitoring standards across production database platforms."

Engineering systems are more valuable than isolated tasks.

---

## Trust Through Evidence

Avoid vague statements.

Support engineering claims using measurable results whenever possible.

Examples

Good

- Managed over 400 production database clusters.
- Led large-scale PostgreSQL migration initiatives.
- Designed reusable migration automation.

Avoid

- Hard worker.
- Fast learner.
- Team player.
- Passionate engineer.

Evidence builds credibility.

---

## Consistency

Every page should feel like part of the same engineering story.

Typography

Spacing

Animations

Components

Tone of voice

All should remain consistent throughout the website.

Consistency communicates engineering discipline.

---

Colors should communicate trust and stability.

Primary colors should remain neutral.

Accent colors should be used sparingly.

Avoid rainbow color palettes.

Avoid excessive gradients.

Typography is the primary visual element.

Large headings should communicate confidence.

- Heavy parallax
- Continuous floating elements
- Excessive motion
- Flashing effects

Motion should feel natural and purposeful.
